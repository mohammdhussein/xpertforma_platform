Auth

POST/api/auth/register/coach/

Public. Register a coach account. New coach accounts start in pending approval status.

Path Params

None

Request Body

{
  "first_name": "Coach",
  "last_name": "Name",
  "phone_number": "0501112222",
  "email": "coach@example.com",
  "password": "StrongPass123!",
  "certificate_image": "coach_certificates/certificate.png"
}

Response 201

{
  "id": "uuid",
  "email": "coach@example.com",
  "approval_status": "PENDING",
  "message": "Registration submitted. Please wait for admin approval."
}

Error Notes

  • 400 if first_name or last_name is missing
  • 400 if certificate_image is blank
  • 400 if the email is already registered

Expected Values

  • first_name, last_name, phone_number, email, password: required strings.
  • certificate_image: required uploaded image/file.
  • approval_status: always PENDING after success.
POST/api/auth/login/

Public. JWT login. Blocks coach accounts that are pending or rejected. Player logins return FIRST_LOGIN or COMPLETE in player_status.login_status.

Path Params

None

Request Body

{
  "email": "user@example.com",
  "password": "StrongPass123!"
}

Response 200

{
  "refresh": "jwt",
  "access": "jwt",
  "access_expires_at": "2026-04-02T18:30:00Z",
  "coach_status": {
    "register_status": "APPROVED"
  },
  "player_status": {
    "has_coach": true,
    "login_status": "COMPLETE"
  }
}

Response 400 (blocked coach)

{
  "coach_status": {
    "register_status": "PENDING"
  }
}

Expected Values

  • email, password: required strings.
  • coach_status.register_status: PENDING, APPROVED, or REJECTED.
  • player_status.login_status: FIRST_LOGIN or COMPLETE.
POST/api/auth/refresh/

Public. Refresh an access token.

Request Body

{
  "refresh": "jwt"
}

Response 200

{
  "access": "jwt",
  "access_expires_at": "2026-04-02T18:30:00Z"
}

Expected Values

  • refresh: required JWT string.
  • access: JWT string.
  • access_expires_at: ISO-8601 UTC datetime string.
POST/api/auth/logout/

Authenticated. Blacklists the current access token JTI in cache and revokes the provided refresh token.

Headers

Authorization: Bearer <access_token>

Request Body

{
  "refresh": "jwt"
}

Response 205

No content

Expected Values

  • Authorization: required Bearer <access_token> header.
  • refresh: required refresh JWT string. After logout it cannot be used at /api/auth/refresh/.
  • Response 205: no body.
POST/api/auth/complete-set-password/

Public. Sets the invited player password using a one-time password setup token from the mobile deep link.

Request Body

{
  "token": "one-time-password-setup-token",
  "password": "NewStrongPass123!"
}

Response 200

{
  "id": "uuid",
  "email": "player@example.com",
  "message": "Password set successfully."
}

Error Notes

  • 400 if token is invalid
  • 400 if token is expired or already used
  • 400 if the token purpose is invalid or its user cannot be resolved
  • 400 if password validation fails

Expected Values

  • token, password: required strings.
  • id: user UUID string.
  • email, message: success strings.

Coach

POST/api/coach/create-player/

Authenticated coach only. Creates a player shell with name, email, and position, reassigns an existing player, and sends a password setup email with a one-time deep link when password setup is required. The rest of the profile is completed later from the player profile endpoint.

Request Body

{
  "name": "Player Name",
  "email": "player@example.com",
  "position_id": 13
}

Response 201 or 200

{
  "id": "uuid",
  "email": "player@example.com",
  "name": "Player Name",
  "position": {
    "id": 13,
    "name": "Center Forward",
    "code": "CF"
  },
  "created": true,
  "password_setup_required": true,
  "setup_password": {
    "deep_link": "https://app.xpertforma.com/set-password?token=...",
    "expires_at": "2026-04-05T14:00:00Z"
  },
  "invitation_sent": true
}

Error Notes

  • 409 if the same coach already has that player email
  • 400 if the email belongs to a coach or manager account
  • 400 if position_id is missing or invalid
  • setup_password is null when the player already has a usable password

Expected Values

  • name, email: required strings.
  • position_id: required integer from /api/positions/.
  • position: object with id, name, code.
  • created, password_setup_required, invitation_sent: booleans.

POST/api/player/checkins/

Authenticated player only. Submits the daily check-in. Returns 409 if a check-in for today already exists.

Request Body

{
  "sleep_hours": 7.5,
  "sleep_quality": "good",
  "mood": 4,
  "sore_zones": ["knee"]
}

Response 201

{
  "id": "uuid",
  "player_id": "uuid",
  "date": "2026-04-16",
  "sleep_hours": "7.50",
  "sleep_quality": "good",
  "mood": 4,
  "sore_zones": ["knee"],
  "readiness_score": 82,
  "created_at": "2026-04-16T06:00:00Z"
}

Expected Values

  • sleep_hours: decimal 0–24.
  • sleep_quality: one of poor, fair, good, great.
  • mood: integer 1–5.
  • sore_zones: array of zone strings (may be empty). Valid values: head_neck, shoulders, upper_back, lower_back, hip_groin, left_quad, right_quad, hamstring, knee, calf, ankle_foot.
  • readiness_score: server-computed integer 0–100.

Error Responses

  • 400: validation error (invalid field values).
  • 409: check-in already submitted for today.
