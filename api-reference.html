Auth

POST/api/auth/register/coach/

Public. Register a coach account. New coach accounts start in pending approval status.

Path Params

None

Request Body

{
  "first_name": "Coach",
  "last_name": "Name",
  "phone_number": "0501112222",
  "email": "coach@example.com",
  "password": "StrongPass123!",
  "certificate_image": "coach_certificates/certificate.png"
}

Response 201

{
  "id": "uuid",
  "email": "coach@example.com",
  "approval_status": "PENDING",
  "message": "Registration submitted. Please wait for admin approval."
}

Error Notes

  • 400 if first_name or last_name is missing
  • 400 if certificate_image is blank
  • 400 if the email is already registered

Expected Values

  • first_name, last_name, phone_number, email, password: required strings.
  • certificate_image: required uploaded image/file.
  • approval_status: always PENDING after success.
POST/api/auth/login/

Public. JWT login. Blocks coach accounts that are pending or rejected. Player logins return FIRST_LOGIN or COMPLETE in player_status.login_status.

Path Params

None

Request Body

{
  "email": "user@example.com",
  "password": "StrongPass123!"
}

Response 200

{
  "refresh": "jwt",
  "access": "jwt",
  "access_expires_at": "2026-04-02T18:30:00Z",
  "coach_status": {
    "register_status": "APPROVED"
  },
  "player_status": {
    "has_coach": true,
    "login_status": "COMPLETE"
  }
}

Response 400 (blocked coach)

{
  "coach_status": {
    "detail": "Coach account is pending approval.",
    "expected": ["APPROVED"],
    "register_status": "PENDING"
  }
}

Rejected coaches receive the same shape with detail = "Coach account has been rejected." and register_status = REJECTED.

Expected Values

  • email, password: required strings.
  • coach_status.register_status: PENDING, APPROVED, or REJECTED.
  • player_status.login_status: FIRST_LOGIN or COMPLETE.

}

Response 200

{

  • access_expires_at: ISO-8601 UTC datetime string.
POST/api/auth/logout/

Authenticated. Blacklists the current access token JTI in cache and revokes the provided refresh token.

Headers

Authorization: Bearer <access_token>

Request Body

{
  "refresh": "jwt"
}

Response 205

No content

Expected Values

  • Authorization: required Bearer <access_token> header.
  • refresh: required refresh JWT string. After logout it cannot be used at /api/auth/refresh/.
  • Response 205: no body.
POST/api/auth/complete-set-password/

Public. Sets the invited player password using a one-time password setup token from the mobile deep link.

Request Body

{
  "token": "one-time-password-setup-token",
  "password": "NewStrongPass123!"
}

Response 200

{
  "id": "uuid",
  "email": "player@example.com",
  "message": "Password set successfully."
}

Error Notes

  • 400 if token is invalid
  • 400 if token is expired or already used
  • 400 if the token purpose is invalid or its user cannot be resolved
  • 400 if password validation fails

Expected Values

  • token, password: required strings.
  • id: user UUID string.
  • email, message: success strings.

Coach

POST/api/coach/create-player/

Authenticated coach only. Creates a player shell with name, email, and position, reassigns an existing player, and sends a password setup email with a one-time deep link when password setup is required. The rest of the profile is completed later from the player profile endpoint.

Request Body

{
  "name": "Player Name",
  "email": "player@example.com",
  "position_id": 13
}

Response 201 or 200

{
  "id": "uuid",
  "email": "player@example.com",
  "name": "Player Name",
  "position": {
    "id": 13,
    "name": "Center Forward",
    "code": "CF"
  },
  "created": true,
  "password_setup_required": true,
  "setup_password": {
    "deep_link": "https://app.xpertforma.com/set-password?token=...",
    "expires_at": "2026-04-05T14:00:00Z"
  },
  "invitation_sent": true
}

Error Notes

  • 409 if the same coach already has that player email
  • 400 if the email belongs to a coach or manager account
  • 400 if position_id is missing or invalid
  • setup_password is null when the player already has a usable password

Expected Values

  • name, email: required strings.
  • position_id: required integer from /api/positions/.
  • position: object with id, name, code.
  • created, password_setup_required, invitation_sent: booleans.

Coach Players

GET/api/coach/players/

Authenticated coach only. Returns the signed-in coach players list.

Query Params

  • q: search by player name, email, or position
  • tab: ALL, ACTIVE, INJURED, or NEEDS_ATTENTION

Response 200

{
  "players": [
    {
      "id": "uuid",
        "name": "Jordan Smith",
        "position": {
          "id": 12,
          "name": "Striker",
          "code": "ST"
        },
      "state": "INJURED",
      "needs_attention": true,
      "expected_return_date": "2026-05-01",
      "avatar_url": "/media/player_avatars/avatar.png"
    }
  ]
}

Expected Values

  • q: optional search string.
  • tab: uppercase only: ALL, ACTIVE, INJURED, NEEDS_ATTENTION.
  • players[].state: stored player state, either ACTIVE or INJURED.
  • players[].needs_attention: server-computed boolean. Injury also counts as needs attention.
  • players[].expected_return_date: YYYY-MM-DD or null.
  • players[]: player objects with position and avatar_url.

Error Responses

Errors for invalid tab include the accepted values in expected:

HTTP 400
{
  "detail": "Invalid tab. Use uppercase values.",
  "expected": ["ALL", "ACTIVE", "INJURED", "NEEDS_ATTENTION"]
}
PATCH/api/coach/players/<player_id>/state/

Authenticated coach only. Updates the stored player state for one of the signed-in coach's players.

Path Params

  • player_id: player user UUID

Request Body

{
  "state": "INJURED",
  "expected_return_date": "2026-05-01"
}

Response 200

{
  "player_id": "uuid",
  "state": "INJURED",
  "expected_return_date": "2026-05-01",
  "needs_attention": true
}

Expected Values

  • state: required uppercase token, either ACTIVE or INJURED.
  • expected_return_date: optional when state=INJURED, format YYYY-MM-DD.
  • Sending expected_return_date with state=ACTIVE returns 400.
  • Setting state=ACTIVE clears the stored expected return date.

Error Responses

HTTP 400
{
  "detail": "Invalid state. Use uppercase values.",
  "expected": ["ACTIVE", "INJURED"]
}
HTTP 400
{
  "detail": "Invalid expected_return_date format.",
  "expected": ["YYYY-MM-DD"]
}

Coach Plans

GET/api/coach/plans/

Authenticated coach only. Returns the signed-in coach's training sessions for an inclusive date range, grouped per day and per plan. Only days with at least one session are returned.

Query Params

  • start_date: optional, format YYYY-MM-DD. Defaults to today.

  • Range must satisfy start_date <= end_date and span at most 31 days.

  "end_date": "2026-04-07",

POST/api/coach/plans/<plan_id>/sessions/<session_id>/start/

Authenticated coach only; the signed-in coach must be the plan creator. Starts the session lifecycle and records present players. Players not in the list are treated as missed once the session is ended.

Path Params

  • plan_id: training plan UUID.
  • session_id: training session UUID; must belong to plan_id.

Request Body

{
  "presentPlayerIds": ["uuid", "uuid"]
}

presentPlayerIds is optional and may be an empty list. Each id must be assigned to the plan.

Response 200

{
  "session_id": "uuid",
  "lifecycle": {
    "status": "IN_PROGRESS",
    "started_at": "2026-04-16T18:00:00+00:00",
    "ended_at": null
  },
  "present_player_ids": ["uuid", "uuid"],
  "missed_player_ids": ["uuid"]
}

Expected Values

  • lifecycle.status: always IN_PROGRESS after a successful start.
  • present_player_ids: sorted UUID strings echoing accepted attendance.
  • missed_player_ids: assigned players not in the present list (sorted UUID strings).

Error Responses

Player id not assigned to the plan:

HTTP 400
{
  "detail": "presentPlayerIds contains players not assigned to this plan.",
  "expected": ["uuid", "uuid"]
}

Other errors:

HTTP 401   unauthenticated
HTTP 403   authenticated user is not the plan creator
HTTP 404   { "detail": "Session not found in this plan." }
HTTP 409   { "detail": "Session has already been started." }

GETPATCHPUT/api/player/profile/

Authenticated player only. Returns the current player profile and updates the player-owned setup fields after password setup. Both PATCH and PUT are accepted. login_status becomes COMPLETE only when the player has an assigned position and all required setup fields are filled; otherwise it remains FIRST_LOGIN.

  "date_of_birth": "2008-04-09",
  "login_status": "COMPLETE",
  "position": {
    "id": 13,

    "code": "CF"

  "team_id": null,

Error Notes

HTTP 400
{

  • avatar_url: always present, returns media URL or null.
  • login_status: FIRST_LOGIN or COMPLETE.
GET/api/player/training/

Authenticated player only. Returns the player's training sessions for an inclusive date range, grouped per day and per plan. Only days with at least one session are returned.

Query Params

  • start_date: optional, format YYYY-MM-DD. Defaults to today.
  • end_date: optional, format YYYY-MM-DD. Defaults to today.
  • Range must satisfy start_date <= end_date and span at most 31 days.

Response 200

{
  "start_date": "2026-04-01",
  "end_date": "2026-04-07",
  "days": [
    {
      "date": "2026-04-01",
      "plans": [
        {
          "plan_id": "uuid",
          "title": "Speed Plan",
          "sessions": [
            {
              "session_id": "uuid",
              "title": "Warm-up",
              "session_type": "GROUP",
              "session_date": "2026-04-01",
              "start_time": "10:00:00",
              "end_time": "10:30:00",
              "intensity": "MEDIUM",
              "location": "Main Field",
              "squad_size": 20,
              "coach_note": "Keep pace moderate",
              "status": "COMPLETED"
            }
          ]
        }
      ]
    }
  ]
}

Expected Values

  • days: sparse list ordered ascending by date; days with no sessions assigned to the player are omitted.
  • plans: ordered by plan title (case-insensitive).
  • sessions: ordered by start_time.
  • session_type: GROUP | TEAM | INDIVIDUAL.
  • intensity: LOW | MEDIUM | HIGH.
  • status: NOT_STARTED | IN_PROGRESS | COMPLETED | MISSED. MISSED appears when the coach has ended the session and the player was not in the present list.
  • start_time / end_time: HH:MM:SS or null. Clients derive duration from these.

Error Responses

Invalid date format includes the accepted format in expected:

HTTP 400
{
  "detail": "Invalid start_date format.",
  "expected": ["YYYY-MM-DD"]
}

Range validation errors:

HTTP 400
{ "detail": "start_date must be less than or equal to end_date." }

HTTP 400
{ "detail": "Date range must not exceed 31 days." }
GET/api/player/sessions/count/

Authenticated player only. Returns per-day session-status counts for the requested inclusive date range (for calendar rendering). Counts are scoped to sessions in plans the player is assigned to. Days with no assigned sessions are zero-filled.

Query Params

  • start_date: required, format YYYY-MM-DD.
  • end_date: required, format YYYY-MM-DD.
  • start_date must be less than or equal to end_date.
  • The inclusive range must not exceed 31 days.

Response 200

{
  "start_date": "2026-04-01",
  "end_date": "2026-04-30",
  "days": [
    { "date": "2026-04-01", "completed_count": 2, "missed_count": 1, "in_progress_count": 2, "not_started_count": 1 },
    { "date": "2026-04-02", "completed_count": 0, "missed_count": 0, "in_progress_count": 0, "not_started_count": 0 }
  ]
}

Expected Values

  • start_date / end_date: echo the requested values in YYYY-MM-DD.
  • days: ordered list covering every day in the inclusive range. Days with no assigned sessions are zero-filled.
  • days[].date: YYYY-MM-DD.
  • days[].completed_count: sessions whose lifecycle is completed and the player has an attendance row.
  • days[].missed_count: sessions whose lifecycle is completed and the player has no attendance row (player was not in the coach's present list).
  • days[].in_progress_count: sessions whose lifecycle is in_progress.
  • days[].not_started_count: sessions with no lifecycle row or with lifecycle not_started.

POST/api/player/checkins/

Authenticated player only. Submits the daily check-in. Returns 409 if a check-in for today already exists.

Request Body

{
  "sleep_hours": 7.5,
  "sleep_quality": "GOOD",
  "mood": 4,
  "sore_zones": ["knee"]
}

Response 201

{
  "id": "uuid",
  "player_id": "uuid",
  "date": "2026-04-16",
  "sleep_hours": "7.50",
  "sleep_quality": "GOOD",
  "mood": 4,
  "sore_zones": ["knee"],
  "readiness_score": 82,
  "created_at": "2026-04-16T06:00:00Z"
}

Expected Values

  • sleep_hours: decimal 0–24.
  • sleep_quality: one of POOR, FAIR, GOOD, GREAT.
  • mood: integer 1–5.
  • sore_zones: array of zone strings (may be empty). Valid values: head_neck, shoulders, upper_back, lower_back, hip_groin, left_quad, right_quad, hamstring, knee, calf, ankle_foot.
  • readiness_score: server-computed integer 0–100.

Error Responses

  • 400: validation error (invalid field values).
  • 409: check-in already submitted for today.

Invalid sleep_quality includes the accepted values in expected:

HTTP 400
{
  "detail": "Invalid sleep_quality.",
  "expected": ["POOR", "FAIR", "GOOD", "GREAT"]
}

Invalid sore_zones includes the accepted zones in expected:

HTTP 400
{
  "detail": "Invalid sore_zones value.",
  "expected": ["head_neck", "shoulders", "upper_back", "lower_back", "hip_groin", "left_quad", "right_quad", "hamstring", "knee", "calf", "ankle_foot"]
}

{
  "id": "uuid",
  "name": "Coach Name",
  "email": "coach@example.com",
  "role": "Coach",

    "approval_status": "APPROVED",

  • coach.approval_status: PENDING, APPROVED, or REJECTED.
